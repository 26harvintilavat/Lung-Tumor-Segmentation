PIPELINE

From raw DICOM to tumor mask
in four steps

A fully automated end-to-end inference pipeline. Drop a ZIP of DICOM files and get segmented overlays back.

01

DICOM Ingestion

Raw CT slices are loaded via pydicom, sorted by ImagePositionPatient, and stacked into a 3D volume.

02

HU Preprocessing

Voxels are converted to Hounsfield Units, resampled to isotropic 1mm spacing, and windowed to the lung preset (−600/1500).

03

Attention U-Net

Three consecutive slices (2.5D context) are fed into the Attention U-Net, which outputs a binary segmentation mask at 256×256 resolution.

04

Overlay & Metrics

Predicted masks are back-projected to original slice dimensions. Tumor regions are highlighted in red and returned as base64 PNG overlays.

ARCHITECTURE

Attention U-Net with
2.5D spatial context

Standard 2D models miss the 3D shape of tumors. We encode depth by stacking three adjacent slices as input channels — giving the network volumetric awareness without the memory cost of 3D convolutions.

2.5D Input Strategy

For each slice z, the model receives [z−1, z, z+1] stacked as a 3-channel tensor. This gives the encoder structural continuity — a vessel that disappears between slices is far less likely to be confused with a tumor nodule.

3 × 256 × 256 input Adjacent context Fewer false positives

Attention Gates

At every skip connection in the decoder, an attention gate learns to suppress activations from irrelevant regions (ribs, blood vessels, airways) and amplify features in suspicious nodule-shaped regions — directly from the loss gradient.

Soft attention Skip connection gating Focal suppression

Hybrid Loss: Tversky + Focal

Lung tumor voxels are vastly outnumbered by healthy tissue. Tversky loss controls the False Negative / False Positive trade-off. Focal loss down-weights easy background pixels and forces the model to focus on hard boundary pixels.

Class imbalance α=0.3 β=0.7 γ=2 focal

Lung ROI Cropping

Before inference, each slice is automatically cropped to a tight bounding box around the lung region. This removes the irrelevant table, body wall, and bed from the input, reducing noise and cutting inference time by ~40%.

DATA

LIDC-IDRI — the gold standard
for lung nodule research

Every model decision was made with reference to the dataset's characteristics: sparse annotations, large class imbalance, multi-rater disagreement.

1,010 CT Scans

From The Cancer Imaging Archive (TCIA). Four radiologists independently annotated each nodule, giving us consensus labels.

XML Annotation Parsing

Raw annotations are in XML format with bounding box + nodule contour data. A custom parser converts these to binary NumPy masks aligned to slice coordinates.

Extreme Class Imbalance

Tumor voxels represent less than 0.5% of all voxels in a typical scan. Standard cross-entropy loss collapses — this is exactly why we use the Tversky + Focal hybrid.

Isotropic Resampling

Slice thickness varies wildly between scanners (0.625mm to 5mm). We resample every volume to isotropic 1mm voxels so the model sees consistent geometry regardless of scanner.

LOSS FUNCTION BREAKDOWN

Tversky Loss α=0.3, β=0.7

Penalises False Negatives (missed tumors) 2.3× more than False Positives. β controls this ratio.

Focal Loss γ = 2

Down-weights correctly classified easy pixels by (1−p)². The model spends its capacity on hard boundary cases.

Combined (0.5 × each) Weighted Sum

Empirically yields higher Dice than either loss alone on LIDC-IDRI validation split.
