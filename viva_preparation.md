# Lung Tumor Segmentation - Viva Preparation Guide

This document covers potential questions and concepts you may be asked during a project presentation or viva for the **LungSeg AI** project.

---

## 1. Project Architecture & Stack

**Q: What is the overall architecture of your project?**
A: The project uses a decoupled client-server architecture.
- **Backend:** A REST API built with Python and `FastAPI`. It handles DICOM file ingestion, uses `pydicom` to extract metadata and pixel arrays, applies Hounsfield Unit (HU) windowing via `numpy`, runs background segmentation pipelines using `FastAPI.BackgroundTasks`, and computes volumetric metrics with `scipy.ndimage`.
- **Frontend:** A Single Page Application (SPA) built purely with Vanilla HTML, CSS, and JavaScript. It uses `Cornerstone.js` for rendering DICOM format images natively in the browser and HTML5 Canvas API for drawing segmentation mask overlays.

**Q: Why did you choose FastAPI over Flask or Django?**
A: FastAPI is highly performant and built on ASGI (Asynchronous Server Gateway Interface). It natively supports `async`/`await` and provides a built-in `BackgroundTasks` feature, which is perfect for long-running machine learning inference pipelines (like tumor segmentation) without blocking the main event thread.

**Q: Why use Vanilla JS instead of React or Vue for the frontend?**
A: To reduce bundle size and external dependencies, demonstrating core DOM manipulation, event routing via hash URLs (e.g., `#upload`, `#viewer`), and direct integration with heavy medical imaging libraries like `Cornerstone.js` without the overhead of Virtual DOM reconciliation.

## 2. DICOM Handling & Medical Imaging

**Q: What is a DICOM file and how do you process it?**
A: DICOM (Digital Imaging and Communications in Medicine) is the standard format for medical images. It contains both pixel data (the scan itself) and rich metadata (patient info, slice thickness, pixel spacing).
In the backend, we strictly use `pydicom` to parse files, extract tags like `SliceThickness` or `PixelSpacing`, and read `PixelData`. In the frontend, `dicomParser` and `cornerstoneWADOImageLoader` are used to render these bytes intelligently.

**Q: Why do you apply Hounsfield Unit (HU) conversion and Windowing?**
A: CT scanners measure radiodensity in Hounsfield Units, scaling from -1000 (air) to +1000 (bone). A typical monitor only displays 256 shades of gray. Windowing maps a specific clinical range of HU values (e.g., Lung Window: Width 1500, Level -600) to the 0-255 grayscale range, highlighting relevant tissues like lung parenchyma and nodules while hiding bones and air. 
We apply this mathematically: `normalized_image = (hu_image - (WL - WW/2)) / WW`.

## 3. Mask Generation & AI Inference

**Q: How is the segmentation mask represented programmatically?**
A: The mask is a 3D binary array outputted by our logic, originally generated by rasterizing XML polygon contours into NumPy arrays (`np.uint8`) where `1` represents a tumor voxel and `0` represents background. This 3D array (`.npy` file) allows us to map each transverse slice precisely.

**Q: How do you compute the Tumor Volume and Max Diameter?**
A: Once we have the 3D binary mask in NumPy:
1. **Tumor Volume:** We sum all voxels with value `1`. We multiply this sum by the **Voxel Volume** (computed as `PixelSpacing[0] * PixelSpacing[1] * SliceThickness`). This gives volume in mm³, which we divide by 1000 to get cm³.
2. **Nodule Count:** We use `scipy.ndimage.label` to find disconnected components in the 3D grid.
3. **Max Diameter:** We iterate over labeled components, find their coordinate bounds, scale them by the pixel spacing, and determine the Euclidean distance across the bounding box.

## 4. Web Interactivity

**Q: How does the "Pipeline Status" polling work?**
A: When a segmentation job is triggered (`POST /segment`), the backend immediately returns a `job_id` and starts the job via a Background Task. The frontend then runs a `setInterval` loop every 2 seconds to `GET /status/{job_id}`, parsing the JSON response to update the visual stepper and append timestamped console logs to the UI until the status marks as "complete".

**Q: How do you overlay the segmentation mask on the CT scan in the browser?**
A: We overlay an absolutely positioned HTML5 `<canvas>` directly on top of the Cornerstone viewport. When jumping to a slice, the frontend fetches the specific mask array for that slice from the backend metrics. We iteratively draw colored pixels (e.g., electric cyan) over the canvas where the array contains `1`, matching the dimensions of the Cornerstone rendered image stream. Alpha transparency controls let the user toggle the overlay opacity.
